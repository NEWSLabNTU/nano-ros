# nros-platform-api

Trait surface defining the platform abstraction layer for nano-ros.
**Internal porting boundary** — user applications depend on
[`nros`](../nros), not on this crate directly.

## Role

This crate is the *single source of truth* for what nano-ros expects
from a platform: a clock, optionally a heap, optionally threading,
optionally networking. Each platform crate
([`nros-platform-posix`](../nros-platform-posix),
[`nros-platform-freertos`](../nros-platform-freertos),
[`nros-platform-zephyr`](../nros-platform-zephyr),
[`nros-platform-nuttx`](../nros-platform-nuttx),
[`nros-platform-threadx`](../nros-platform-threadx),
the bare-metal crates under [`packages/platforms/`](../../platforms))
implements the subset its target supports.

## Source layout

| Module | Trait family | Required by |
|--------|--------------|-------------|
| `clock` | `PlatformClock` | every backend |
| `alloc` | `PlatformAlloc` | zenoh-pico |
| `sleep` | `PlatformSleep` | zenoh-pico |
| `random` | `PlatformRandom` | zenoh-pico |
| `time` | `PlatformTime` | zenoh-pico |
| `threading` | `PlatformThreading` (tasks + mutex + condvar) | multi-threaded backends |
| `network_poll` | `PlatformNetworkPoll` | bare-metal smoltcp |
| `tcp` | `PlatformTcp` | zenoh-pico TCP transport |
| `udp` | `PlatformUdp` | XRCE-DDS, dust-DDS, zenoh-pico UDP transport |
| `udp_multicast` | `PlatformUdpMulticast` | RTPS SPDP discovery, zenoh scouting |
| `socket_helpers` | `PlatformSocketHelpers` | zenoh-pico transport layer |
| `libc` | `PlatformLibc` | bare-metal targets without a C runtime |

The trait grouping (and the per-method behaviour contract — blocking,
may-fail, unsupported-fallback) is documented at length in the
[Platform API Design](../../../book/src/design/platform-api.md) page.

## When to use

- Implementing a new platform port — implement these traits on a
  zero-sized type, then wire the type into
  [`nros-platform`](../nros-platform) via a `platform-<name>`
  Cargo feature.
- Reading the C vtable (`nros-platform-cffi`) — the C `nros_platform_vtable_t`
  is auto-generated by cbindgen from the Rust trait family in this
  crate, so the C surface and Rust never drift.
