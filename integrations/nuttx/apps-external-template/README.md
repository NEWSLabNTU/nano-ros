# NuttX `apps/external/<bringup>/` template (Phase 212.H.2)

This directory is the **per-bringup adapter shim** that NuttX's
`apps/external/*/Make.defs` discovery glob picks up when symlinked (or
copied) into `$NUTTX_APPS_DIR/external/<bringup>/`. It is the canonical
embedded entry point for the Phase 212 `<system>_bringup` package shape
on the NuttX target.

## Files

| file | role |
|---|---|
| `Make.defs` | Appends `apps/external/<bringup>` to `CONFIGURED_APPS` when `CONFIG_NROS_BRINGUP_DEMO=y`. |
| `Makefile`  | Defines `context::` (runs `nros codegen-system` + cross `cargo build`) and `clean::`. |
| `Kconfig`   | Exposes a single `bool NROS_BRINGUP_DEMO` knob under External Modules → nano-ros bringup. |
| `nros_bringup.mk` | (Generated by `stage-external-apps.sh --bringup`) Pins `NROS_BRINGUP_NAME` + `NROS_BRINGUP_WORKSPACE`. |

The shim stays ≤200 LoC (`tokei` budget in `docs/design/0003-rtos-integration-pattern.md` §6).

## Deploy by symlink (recommended)

```sh
# Symlink the workspace's bringup into NuttX apps/external/. The staging
# script wires per-bringup pinning + regenerates apps/external/Kconfig.
scripts/nuttx/stage-external-apps.sh "$NUTTX_APPS_DIR" \
    --bringup "$WORKSPACE/src/demo_bringup"

# Then configure + build NuttX as usual.
cd "$NUTTX_DIR"
make menuconfig   # Application Configuration → External Modules → nano-ros bringup → enable
make
```

Manually:

```sh
ln -s "$WORKSPACE/src/demo_bringup" "$NUTTX_APPS_DIR/external/demo_bringup"
cp /path/to/integrations/nuttx/apps-external-template/{Make.defs,Makefile,Kconfig} \
    "$NUTTX_APPS_DIR/external/demo_bringup/"
```

## Copy-out (offline / vendor-frozen NuttX trees)

```sh
scripts/nuttx/stage-external-apps.sh "$NUTTX_APPS_DIR" \
    --bringup "$WORKSPACE/src/demo_bringup" --copy
```

## What the `context::` rule does

1. Runs `nros codegen-system --bringup <name> --target nuttx --out
   <build>/nros-system/` — host-time bake of `system.toml` +
   `launch/*.xml` → `system_config.h` + `system_main.c` consumed by the
   bringup Rust crate (`nuttx_app_glue/`).
2. Runs `NROS_CARGO_BUILD` (the macro from the shared
   `integrations/nuttx/Makefile`) — cross-builds the bringup workspace
   for `armv7a-nuttx-eabihf` (or whichever NuttX target triple the
   `qemu-armv7a/nsh` defconfig resolves to).

The resulting staticlib is appended to `EXTRA_LIBS` and linked into the
monolithic NuttX kernel ELF.

## Pre-requisites

- `nros` CLI on `PATH` (or `$NROS_BIN` set), with Phase 212.E
  `codegen-system` support (otherwise the `context::` rule errors with
  the verb-not-found message — build via `just setup-cli` +
  `source ./activate.sh` from the nano-ros root, Phase 218).
- The shared `integrations/nuttx/` shell symlinked at
  `$NUTTX_APPS_DIR/external/nano-ros/` (the staging script does this
  unconditionally).

See `docs/design/0024-multi-node-workspace-layout.md` for workspace shape and
`docs/design/0003-rtos-integration-pattern.md` §3 (NuttX row) for the
end-to-end build flow.
