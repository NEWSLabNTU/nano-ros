# ws-custom-msg-mixed — C node packages under a C++ typed entry

## What it shows

Language mixing at the workspace level: the **C** `reading_talker` /
`reading_listener` node packages are reused **verbatim** from
[`ws-custom-msg-c`](../ws-custom-msg-c/) (same `custom_msgs/Reading.msg`, same
`/reading` topic), but the entries are generated by the **C++ TYPED entry
carrier** (`nano_ros_entry(… TYPED)` in `native_talker_entry` /
`native_listener_entry`). Two processes (issue 0096).

## Run

```sh
source ./activate.sh && nros ws sync
nros codegen-system --bringup demo_bringup
cmake -S . -B build && cmake --build build
zenohd --listen tcp/127.0.0.1:7447 &
./build/src/native_listener_entry/native_listener_entry &
./build/src/native_talker_entry/native_talker_entry
```

## Expected output

Same as the C workspace:

```
[reading_talker] sent seq=1 temp=21.5
reading seq=1 temp=21.5
```

## Copy-out notes

Standard workspace copy-out. Fixture ids
`workspace-mixed-native-custom-msg-{talker,listener}`.
