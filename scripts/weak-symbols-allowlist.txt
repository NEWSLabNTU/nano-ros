# Issue 0050 — audited weak-symbol allowlist (single source of truth).
#
# Every owned C/C++/asm file that defines weak symbols (`__attribute__((weak))`
# / `.weak`) is listed here with its expected weak-decl count + classification.
# Both gates read this file:
#   - scripts/check-weak-symbols.sh  (fast source-level scan, in `just check`)
#   - nros-tests/tests/weak_symbol_audit.rs (the same scan as a Rust test)
# Drift (a new weak site, or a count change in a listed file) fails both —
# forcing a re-audit + an update here.
#
# Format:  <expected-count> <repo-relative-path>   # classification
# (vendored zenoh-pico / mbedtls / third-party are excluded from the scan.)
# Classification: override-default = a strong def is guaranteed elsewhere
# (board / app / cmake stub); optional-hook = the weak no-op IS the intended
# fallback.
#
# Image gate (scripts/check-weak-symbols-image.sh) SSoT: an override-default
# line carries an `[img: <sym> <sym> …]` token naming the exact symbols whose
# strong override must win in a FINAL linked image. That gate cross-checks its
# coverage map against this set: every covered symbol must be declared here,
# and a declared symbol with no coverage row yet is reported (not silently
# dropped). Both source-level gates take only the text before `#`, so the
# token is invisible to them. optional-hook lines carry no token (the weak
# no-op is a valid runtime — nothing to assert in an image).
