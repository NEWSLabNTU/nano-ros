# wcr bundle layout вЂ” v0.1

**Status:** Frozen at Phase 211.4 (2026-06-01). The format never breaks
within v0.1.x; additive extensions land via minor schema version bumps.
Breaking changes require a major bump.

A wcr bundle is a `.tar.zst` archive containing a verified package's
source (optional, tier-dependent), proof artifacts, attestations, and
SBOM. One bundle per `(crate, version)` pair. Filename convention:

```
<crate>-<version>.wcr.tar.zst
```

## File-by-file specification

### `manifest.toml`

A verbatim copy of the package's `[package.metadata.nros.proof]` table
(or the `[proof]` section of a non-Cargo `wcr.toml`). Validates against
`proof-section-v0.1.json`. Top-level table only вЂ” Cargo-specific keys
(`package.name`, `package.version`) are repeated here for self-contained
identification:

```toml
[package]
name    = "nros-cmd-gate"
version = "0.4.1"

[proof]
schema_version = "0.1"
kind           = "code"
tier           = 1
language       = "rust"
# ... rest of the [package.metadata.nros.proof] body verbatim ...
```

### `source/`

Optional. Present when `source.lock.embedded = true` (Tier 0-1 default;
opt-out at Tier 2-3 proprietary mode in 211.8+). Mirrors the package
source tree under the bundle root. Files included per the
`source_discovery` rules recorded in `source.lock`.

Default include / exclude:

- Include: `**/*.rs`, `**/*.c`, `**/*.h`, `**/*.cpp`, `**/*.hpp`,
  `Cargo.toml`, `wcr.toml`, `CMakeLists.txt`, `build.rs`, `proofs/`
- Exclude: `target/`, `.git/`, `*.o`, `*.a`, `*.so`, build outputs

Authoring CRATE may override via `[package.metadata.nros.proof.source_discovery]`
keys.

### `source.lock`

Always present. JSON, validates against `sourcelock-v0.1.json`. Carries
the canonical-source URI, Merkle tree hash, embed flag, file count, and
total bytes.

When `embedded = false`, consumers fetch the source via `source_uri`
and recompute the Merkle root to verify byte-identical match.

### `proofs/axioms/`

Author-maintained Why3 axiom theories. Present only when the package
declares `local_axioms` in its manifest. Each file is a self-contained
Why3 theory module.

Naming convention: `<dep>_axioms.mlw` or `<topic>_axioms.mlw`.

### `proofs/ghosts/`

Author-maintained Why3 ghost theories. Present only when the package's
proofs reference ghost state. Each file is a Why3 theory module
declaring abstract types + functions invisible to the consumer's impl
but referenced in the spec.

### `proofs/extracted/<lang>/<theory>.mlw`

Language-keyed extracted Why3 theories. Generated by the extractor:

- `extracted/rust/` в†ђ Creusot output for Rust crates
- `extracted/c/` в†ђ Frama-C/WP output for C crates (post-PoC, 211.8+)

One file per emitted theory. Theory names follow the convention
`<Crate>_<module>` (PascalCase, `-` в†’ `_`).

This IS the spec вЂ” the artifact consumers import via Why3 `use`. No
hand-authored sibling spec file exists (M3 / M-no-spec policy).

### `proofs/session/<lang>/`

Language-keyed Why3 session bookkeeping:

- `why3session.xml` вЂ” per-VC discharge records (prover, time, transformations)
- `why3shapes.gz` вЂ” goal-shape cache for cross-version matching

Replayable via `wcr replay`. Cache hits on shape match across minor
edits to the extracted file.

### `proofs/coq/<vc-id>.v`

Optional (M4 вЂ” Coq fallback workflow). Present only when one or more
VCs required a manual Coq tactic script. Replayed via Why3's Coq
driver during `wcr replay`.

### `attestations/proof.intoto.jsonl`

In-toto v1 attestation in JSONL format. One Statement per line, one
Statement per `(crate, language)` pair (PoC ships one line; future
cross-language packages append C lines under `extracted/c/`).

`predicateType` = `https://github.com/NEWSLabNTU/wcr/proof/v0.1`
(provisional GitHub-path URI in PoC; may swap to wcr.dev when domain
registered).

Predicate body validates against `attestation-predicate-v0.1.json`.

### `attestations/slsa.intoto.jsonl`

In-toto v1 SLSA build provenance statement. Records the build pipeline,
materials, and outputs. Issued by the build runner (CI or local).

### `attestations/sigstore.bundle`

Post-PoC (211.8). Sigstore-shaped signing bundle covering all
attestations. Present at Tier в‰Ґ 1 once the registry is hosted.

### `sbom.cdx.json`

CycloneDX SBOM in JSON format. Lists every dep вЂ” code + axiom вЂ” with
versions, hashes, and license identifiers. Generated by the build
pipeline; validated by `wcr bundle`.

## Archive format

- Compression: zstandard, level 19 (`--ultra` off вЂ” balances ratio vs
  build-time)
- Tar block size: default (10240 bytes)
- File ordering inside tar: lexicographic on path (reproducible)
- Mode bits stored: all 644 (data) and 755 (executable) only; no other
  permissions
- Timestamps stored: SOURCE_DATE_EPOCH if set, else 0 (reproducible)

Reproducibility: two `wcr bundle` runs on identical inputs (same source
tree, same extracted output, same session XML) produce byte-identical
bundles.

## Size budget

| Crate class | Typical bundle size |
|---|---|
| Tiny axiom crate (single theory, no source) | в‰¤ 32 KB |
| Code crate (Tier 1, embedded source ~50 files) | в‰¤ 1 MB |
| Code crate with vendored C deps (e.g. RMW) | в‰¤ 10 MB |
| Tier-3 cert kernel (with Coq scripts + ghosts) | в‰¤ 20 MB |

If a bundle exceeds 50 MB, `wcr bundle` emits a warning + suggests
checking `source_discovery` excludes.

## Forward-compat invariants

The following bundle properties never change within v0.1.x:

- Top-level directory names (`source/`, `proofs/`, `attestations/`)
- Language-keyed `extracted/<lang>/` + `session/<lang>/` hierarchy
- File naming conventions (`*.mlw`, `*.intoto.jsonl`, `*.cdx.json`)
- `manifest.toml` at root
- `source.lock` at root
- In-toto Statement-per-line JSONL semantics
- Reproducible tar entry ordering + zstandard compression

Adding new file classes (e.g. ELPI traces, alternative SMT
certificates) lands under a new directory namespace; existing layout
unchanged.

## Compatibility statement

A v0.1 consumer is guaranteed:

1. To read any v0.1.x bundle produced by any wcr 0.1.x release.
2. To produce a v0.1 bundle consumable by any v0.1.x wcr consumer.
3. To error explicitly (exit code 8) on hash mismatch rather than
   silently re-processing.

Cross-version bundles (e.g. v0.2 в†’ v0.1 consumer) are out of scope
until a v0.2 lands.
